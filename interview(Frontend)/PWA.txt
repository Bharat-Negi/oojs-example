Fundamentals:-
Q1. What is a PWA?
A: A Progressive Web App (PWA) is a web application that leverages modern web capabilities to deliver an app-like experience. It can work offline, be installed on a user‚Äôs device, send push notifications, and load instantly regardless of network quality. Unlike native apps, PWAs run in the browser but can still provide native-like features.

+--------------------------------------------------------------------------------------------------------+
Q2. What are the core principles of a PWA?
A: Google defines three key principles:
‚Ä¢	Reliable ‚Üí Loads instantly and works offline or on poor connections.
‚Ä¢	Fast ‚Üí Responds quickly to user interactions.
‚Ä¢	Engaging ‚Üí Feels like a native app, can be installed, and re-engages users with push notifications.

+--------------------------------------------------------------------------------------------------------+
Q3. What technologies enable PWAs?
A:
‚Ä¢	Service Workers ‚Üí Offline support and caching.
‚Ä¢	Web App Manifest ‚Üí Metadata that enables installability.
‚Ä¢	HTTPS ‚Üí Secure environment required for service workers and APIs.
‚Ä¢	Modern APIs ‚Üí For push notifications, geolocation, media, etc.

Service Workers:-
Q4. What is a Service Worker, and why is it important?
A: A Service Worker is a script that runs in the background, separate from the main browser thread. It intercepts network requests and decides whether to fetch from the network or serve cached content. It‚Äôs crucial for offline mode, background sync, and push notifications.

‚Ä¢	Network First ‚Üí Best for dynamic data (APIs).
‚Ä¢	Stale-While-Revalidate ‚Üí Serves cached content while updating in the background.
‚Ä¢	Cache Only ‚Üí Useful for fully offline apps.

Performance & Reliability
Q8. How do PWAs improve performance?
A: They cache assets and content locally, reducing dependency on the network. For example, first visits may fetch data from the server, but subsequent visits can load instantly from cache. Techniques like lazy loading, prefetching, and image optimization further enhance performance.

+--------------------------------------------------------------------------------------------------------+
Q9. How do you measure and optimize PWA performance?
A:
‚Ä¢	Tools: Google Lighthouse, Chrome DevTools, WebPageTest.
‚Ä¢	Metrics: Core Web Vitals (LCP, CLS, FID), Time to Interactive, First Contentful Paint.
‚Ä¢	Optimizations: Minify resources, use HTTP/2, compress assets, adopt proper caching strategies.

Native-Like Features
Q10. How do PWAs support offline mode?
A: Service Workers cache essential files. If the network fails, the Service Worker serves cached responses. Many PWAs also provide a custom offline fallback page to handle missing network requests gracefully.

+--------------------------------------------------------------------------------------------------------+
Q11. How do push notifications work in PWAs?
A:
1.	User grants notification permission.
2.	Browser generates a subscription object via the Push API.
3.	Server sends a push message using Web Push protocol and VAPID keys.
4.	Service Worker receives the push event and displays the notification.

Security
Q12. Why must PWAs run on HTTPS?
A: Service Workers can intercept network requests, so HTTPS ensures data integrity and security, preventing man-in-the-middle (MITM) attacks. It also secures sensitive APIs like Geolocation and Push.

+--------------------------------------------------------------------------------------------------------+

‚Ä¢	Use IndexedDB with encryption if needed.
‚Ä¢	Always use secure authentication (OAuth, JWT).
‚Ä¢	Prefer server-side validation for critical actions.

Advanced Concepts
Q16. What is Background Sync in PWAs?
A: Background Sync lets PWAs defer actions until network connectivity is restored. For example, a user adds a comment offline, and the Service Worker syncs it automatically when back online.
